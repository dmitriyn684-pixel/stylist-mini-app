# План первых 100 лидов

## Правило набора

100 лидов — research pool, а не список для массовой рассылки. Контактировать только после квалификации. На каждого лида зафиксировать источник, наблюдение, одну боль, один оффер и следующий шаг.

## Lead score 0–10

- `+2` есть активный продукт или вакансия.
- `+2` видна конкретная задача, которую закрывает профиль.
- `+2` понятен decision maker или штатный путь отклика.
- `+2` можно сослаться на релевантный кейс.
- `+1` контакт свежий: публикация/вакансия активна.
- `+1` совпадают рынок, язык, формат и доступность.

Контактировать сначала с оценкой 7–10. Лиды 4–6 оставить на дополнительный research; 0–3 не контактировать.

## CRM status reference

`new` → `qualified` → `message_ready` → `contacted` → `replied` → `call_booked` → `deck_sent` → `proposal_sent` → `won`.

Альтернативные финальные или отложенные статусы: `lost`, `not_relevant`, `follow_up_later`. Статус описывает текущий этап, а не качество человека. После каждого касания обновлять `last_contact_at` и `next_follow_up_at` в ISO-формате `YYYY-MM-DD`.

## Порядок сбора

1. Создать 10 строк из одной группы и проверить качество.
2. Удалить дубли и неактуальные контакты.
3. Написать персональное наблюдение и выбрать материал.
4. Только после этого добавить следующие 10.
5. Сохранять причину отказа или отсутствия fit, а не только статус.

## Definition of done

- 100 уникальных строк.
- У каждой строки есть публичный source URL и дата проверки.
- Минимум 60 лидов имеют score 7+.
- Для каждого score 7+ подготовлено индивидуальное первое сообщение.
- Нет скрытых личных данных, автособранных контактов и массовой отправки.
